Tutorial Git e GitHub - Gustavo Nery

  1. Instale o Git e faça uma conta no GitHub;
  2. Abra o Git Bash;
  3. Escreva git clone <URL do repositório>;
  4. Abra a pasta criada e crie seus documentos;
  5. No terminal, digite os seguintes passos:
    1. git init
    2. git add .
    3. git commit -a -m "nome do commit"
    4. git log
    5. git remote origin <URL do repositório>
    6. git push

Funções Git

  • git init - Inicializa um novo repositório Git na pasta atual.
  • git log - Mostra o histórico de commits no repositório.
  • git reflog - Exibe o registro de referências.
  • git add . - Adiciona todas as mudanças na área de preparação (staging area).
  • git add <nome do arquivo> - Adiciona um arquivo específico na área de preparação.
  • git commit -a -m "nome do commit" - Realiza um commit com todas as mudanças e adiciona uma mensagem.
  • git commit <caminho do arquivo> - Realiza um commit de um arquivo específico.
  • git branch - Lista todas as branches e mostra a branch atual.
  • git checkout - Permite alternar entre branches ou commits específicos.
  • git branch <nome> - Cria uma nova branch.
  • git pull - Atualiza o repositório local com as mudanças do repositório remoto.
  • git clone <URL> - Clona um repositório remoto para o seu sistema local.
  • git push - Envia os commits locais para um repositório remoto.
  • git merge <nome da branch> - Combina os commits de uma branch com a branch atual.
  • git remote -v - Exibe os repositórios remotos configurados para o repositório local.
  • git status - Mostra o estado atual do seu diretório de trabalho.
  • git tag - Gerencia tags (etiquetas) para marcar pontos específicos na história do repositório.
  • git log --graph - Exibe o histórico de commits de forma gráfica.

Commits Semânticos

Os commits semânticos seguem um padrão de mensagem que expressa a natureza das mudanças realizadas. Exemplo:

  • feat: - Adiciona uma nova funcionalidade
  • fix: - Corrige um bug
  • docs: - Atualiza a documentação
  • style: - Faz ajustes de estilo
  • refactor: - Refatora o código
  • test: - Adiciona ou modifica testes
  • chore: - Realiza tarefas de manutenção ou outras pequenas alterações

Para mais detalhes, consulte: Conventional Commits
